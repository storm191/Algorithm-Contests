Problems

3. Copy Cat

CopyCat is a primitive 2-bit turing machine like processor. It has 2 tapes, a read-only input tape and a non-rewritable output tape. It has an input tape which is initially filled with some data. Each cell in the input tape can contain a 2-bit value, viz., 0, 1, 2 or 3. The machine has only one instruction called "COPY". The instruction has the following syntax:

COPY [INPUT|OUTPUT] [POS1]:[POS2] [POS3]

This instruction copies cells from POS1 to POS2 in the INPUT or OUTPUT tape (as specified in the instruction) into locations POS3 to POS3+|POS2-POS1|+1 on the output tape. Note that of POS1>POS2, the cells are copied in reverse on the output tape. Initially the values in the output tape are set to UNUSED. Once a value is written into the output tape, you cannot rewrite it. Given an input tape configuration, you are required to find the minimum number of COPY instructions to obtain the required output configuration, if possible.

Example: Input: 0123 Output: 32010022002 The instructions involved are

  1. COPY INPUT 4:3 1
    Output: 32---------
  2. COPY INPUT 1:2 3
    Output: 3201-------
  3. COPY OUTPUT 2:3 8
    Output: 3201---20--
  4. COPY OUTPUT 9:8 10
    Output: 3201---2002
  5. COPY OUTPUT 10:8 5
    Output: 32010022002

Five operations are required.

Input

The first line is T the number of test cases. Then T test cases follow. Each case has two lines, the first line has the input string, the second the output string required. Both strings are made up of the characters 0,1,2 and 3.

Output

The output has T lines. Line i (1<=i<=N) has the minimum number of instructions required for case "i". If this is not possible, the line should contain the string "impossible".

Constraints

1<=T<=100
Input and output string lengths are less <=18.

Sample input

5 0123 2301 0 1 0123 3210 0123 31203120 0 000000000000000000

Sample output

2 impossible 1 4 6

Time limit: 2 seconds

Memory limit: 64 MB
